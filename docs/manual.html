メリット

Simple Motion Tracker（以後SMT）を使うメリットは、バーチャルモーションキャプチャー（以後VMC）とVMCの周辺ツールの恩恵を受けられることです。

すでにVR機器でVMCやEVMC4Uを使いリッチな絵作りをされている方は、VR機器をSMTに置き換えるだけでクオリティを維持しつつWebカメラでの気軽な撮影や配信が可能になります。

初めてVMCを利用される方は、SMTを使いVMCでの撮影や配信環境を整えておくことで、将来的にVR機器導入した際には、SMTをVR機器に置き換えるだけで移行が行えます。



すぐに始める
１．アプリケーションの入手

Simple Motion Traker と Virtual Motion Capture ダウンロードします。


２．Simple Motion Trakerの起動と設定

①ドロップダウンメニューからカメラを選択する（※既知の不具合、仮想カメラがあると異なるカメラが選ばれることがある）

②Startを押す（カメラが起動したか確認したい場合は画面下部の Show Capture Imageにチェックを入れる）

③Use Face Trackingにチェックを入れる

④Calibrate Face Pointsを押す

⑤Force T Poseにチェックを入れる


３．Virtual Motion Captureの起動と設定

①詳細設定を開く

②OSCでモーション受信を有効にするにチェックを入れる

③トラッカー割り当て設定を開くを押す

④トラッカー一覧に以下の仮想トラッカーが表示されていることを確認する

  • ・VD_SMT_HEAD
  • ・VD_SMT_LEFT_HAND
  • ・VD_SMT_RIGHT_HAND

⑤トラッカー割り当てで頭、左手、右手に仮想トラッカーを割り当てる

トラッカー割り当て設定、設定のウインドウを閉じる


４．アバターの読み込みとキャリブレーション

①VRM読込から[VRM読み込み]→[ファイルを選択]→[同意して読み込み]でアバターをロードする

②キャリブレーションを開く

③モードを選びキャリブレーションを実行する



５．トラッキングスタート！

①SMTのForce T Poseのチェックを外すとTポーズが解除され、顔の動きに合わせてVMC上のアバターが動きます




画面詳細
各パラメータの解説
Foce T Pose
VMCでのキャリブレーション用にTポーズを取ります

Use Face Tracking
顏認識による追跡モードを有効にします

Translation Magnification
平行移動量の倍率です。値を大きくすると少しの頭の動きでアバターを大きく移動させることが出来ます（デフォルトは1.0）
１つ目は全ての軸の倍率に影響します。２～４個目はそれぞれX, Y, Z軸に影響します。

Rotation Magnification
回転量の倍率です。値を大きくすると少しの頭の動きでアバターを大きく回転させることが出来ます（デフォルトは1.0）
１つ目は全ての軸の倍率に影響します。２～４個目はそれぞれX, Y, Z軸に影響します。

Calibrate Face Points
顏の中央位置、奥行きの最背面の位置確定させます。この座標をもとに顏の傾きや前後を検出します。また、この位置より奥へは移動できません。
※後述のAuto adjustmentにより補正されます。

Use Eye Tracking
目の追跡モードを有効にします

Use Blink
Use Eye Trackingが有効の時、目の開閉を利用します。
※バーチャルモーションキャプチャー側の自動まばたきをOFFにしてください。

LR Sync
Use Eye Trackingが有効の時、両目の開閉を同期します。（両目の平均値を取る）

Iris Threshold
虹彩（黒目）の閾値を0～255で指定します。閾値以下の暗さの部分を虹彩として処理します。

Iris Offset
虹彩のずれを補正します。X, Y軸それぞれ-1～1の間で指定します。

Iris Translation Magnification
虹彩の平行移動量の倍率です。値を大きくすると機敏に反応します。X, Y軸でそれぞれ指定します。

Eye Open Threshold
目の開閉の閾値の最小と最大を設定します。最小以下は全て閉じ、最大以上は全て開き、として扱われます。

Use Hand Tracking
手の追跡モードを有効にします。

Hand Moving Threshold
手の動体検知面積の閾値です。最小、最大を指定します。
最小：検知が最小を下回ると追跡を停止します。(デフォルトは0.05)
最大：動き始めはこの値を超える量の動体検知面積が必要です。(デフォルトは3.0)

Hand Undetected Duration
手の動体検知を停止してから完全にリセット状態になるまでの待機時間です。この時間を超えると手を下します。

Hand Offset
手の位置のオフセットをX,Y,Z軸で指定します

Hand Translation Magnification
手の平行移動量の倍率です。X,Y,Zそれぞれの軸を指定します。(デフォルトは1.0、1.0、0.3)
値を大きくすると少しの動きで手を大きく動かすことが出来ます

Smoothing Level
検出誤差を抑えるために指定フレーム間の平均値を取ります。値が小さいとクイックに反応しますが、ブレが大きくなります。値が大きいとブレは少ないですが反応が遅くなります。（デフォルトは10）

Select Video Device
カメラの選択です。カメラを変更すると動いていたカメラが停止します
※既知の不具合。仮想カメラがあると（特にFaceRigが怪しい）指定したカメラが選ばれないので上から順に試してもらう必要があります。

Start
選択したカメラに切り替えます。押した後は表示がStartからRunningに切り替わります

Mirror
動きを左右反転させます。

Port
仮想トラッカー情報をVMCへ送るためのポート番号です

Show Capture Image
カメラで撮影中の現実が表示されます。カメラで撮影できているかの確認や顏の検出状況の確認が出来ます

Save / Load
パラメータの設定情報をファイルに保存、ファイルから読み込みます。

VR Play Area Offset Translation / VR Play Area Offset Rotation
VR機器のプレイエリアとSMTの座標を合わせるための平行移動、回転のオフセットです。
位置を合わせることでVR機器とSMTの混成が可能になります。(頭はSMT、手はVRコントローラーなど)

Load Bone VRM
指をコントロールする際にVMCに読み込んだアバターと同じボーン構造のVRMを読み込みます。
読み込みが完了すると Status: に "VRM is loaded" と表示されます。
読み込んだ状態でSMTの設定ファイルを保存しておくと、次回設定ファイルを読み込んだ際にVRMが自動的にロードされます。

Use Finger Control With Input Devises
キーボード、マウス、ジョイパッド(ゲームパッド) の入力により指の曲げ伸ばしを制御します。

Use Grip Poses
デフォルトポーズとは別に、ジョイパッドなどを握り込んだようなポーズを取らせる事ができます。

Select Joypad Device
ジョイパッド(ゲームパッド)を選択します。
切り替えを適用するには "Use Finger Control With Input Devises" のチェックを一度OFFにして再度ONにする必要があります。
※Ver0.5 では DirectInput に対応したデバイスのみ認識します。



逆引きリファレンス
設定の保存と読み込み
チェックボックスのON/OFFや設定した数値、カメラやジョイパッドの選択情報をファイルに保存することができます。
※ファイルパスに全角文字が含まれていると正常に読み込めない場合があります。

カメラの選択と起動
利用したいカメラを選択します。
※既知の不具合、利用したいカメラより上に仮想カメラが表示されている場合、仮想カメラの数だけ選択位置がずれて想定外のカメラが起動される場合があります。
以下の画像の例では「Logicool HD Webcam」を起動するには「FaceRig Virtual」を選択する必要があります。
 カメラ選択後にStartボタンを押すとカメラが起動します

カメラが正常に起動できているか確認します。Show Capture ImageのチェックをONにすると、別ウインドウでカメラの映像が表示されます。

顏の位置と向き
カメラを起動した後に Use Face Tracking をONにすると顏のトラキングが開始されます。
顏の基準位置を設定するために Calibrate Face Position ボタンを押します。

Translation Magnification(TM)とRotation Magnification(RM)は移動と回転で実際の動きに対して倍率をかけることができます。
例えばTMのALLに2倍を設定すると、上下左右前後に大きくアバターが動きます。
またRMのX,Yに2倍を設定すると上下左右の顏の向きが大きくなります。

視線とまばたき

①Use Eye Tracking をONで視線の動き、②Use Blink をONでまばたきのトラッキングが行われます。
カメラのキャプチャ映像で瞳に赤い〇が表示されていれば視線のトラッキングが行われています。
もし赤い丸が瞳に対して大きすぎる場合は③Iris Thresholdの値を小さくしてください。逆に小さすぎる場合は値を大きくしてください。（おおよそ20～60が適正範囲です）
正面を向いているのにアバターの視線が上下左右にずれる場合は④Iris OffsetでX,Yそれぞれの方向に-1～1の範囲で調整をかけることができます。
アバターの視線の動く量を調節したい場合は⑤Iris Translation Magnification の倍率を調整してください。X,Y方向を個別に指定することが出来ます。

まぶたの開閉がうまく認識されない場合は⑥Eye Open Thresholdで全閉じ、全開きの閾値を設定することができます。
目を開いているのにアバターの半目になってしまう場合は最大値の値を下げてください。
逆に目を閉じているのにアバターの半目になってしまう場合は最小値の値を上げてください。



簡易的な手の位置

①Use Hnad TrackingをONにすると手のトラッキングが行われます。
カメラのキャプチャ映像で手の動きに合わせて緑の〇が表示されていれば手のトラッキングが行われています。
手を動かしても緑の〇が出ない場合は②Hand Moving Thresholdの最大値を下げてください。

手をカメラの範囲外にだすなど、SMTが手を追跡できなくなった場合には自動的に手を下げます。
手を下げるまでの時間は③Hand Undetected Durationで指定することができます。

手の位置は顏の位置を元に計算されます。表示位置をずらしたい場合は④Hand OffsetでX,Y,Zそれぞれの軸に対して調整することができます。

カメラの撮影範囲が狭い場合や、実際の手の動きとアバターの動き(腕の伸びなど)が一致しない場合などで、手の移動量を調整したい場合は⑤Hand Translation MagnificationでX,Y,Zそれぞの軸に対して倍率を設定することができます。



入力機器での指の曲げ伸ばし

入力デバイスによる指の曲げ伸ばしを行うにはVMCに以下の設定が必要です。
VMCの詳細設定にある "OSCモーション受信を有効にする" と "手ボーン取得" をONにします。

SMT側では Load Bone VRM ボタンを押してVMCに読み込んでいるVRMと同じボーンを持つVRMを読み込みます。
読み込みに成功した場合 Status: の所に VRM is loaded. と表示されます。

Use Finger Control With Input Devises をONにすると、入力でデバイスのキーやボタン押下に合わせてアバターの指が動きます。
Select Joypad Device で有効なジョイパッド(ゲームパッド)を選ぶことが出来ます。(Ver0.5ではDirectInputに対応したデバイスのみ動作します)
デバイスの変更は Use Finger Control With Input Devisesを OFF/ONすることで適用されます。
Use Grip PosesをONにするとジョイパッドを握り込んだようなポーズを取らせることができます。

TIPS
フェイストラッキングと併用するとフェイストラッキングの負荷に影響を受けて指の動きの滑らかさが失われます。
併用したい場合は、SMTをもう一つ立ち上げて、顏目手に関するチェックボックスをすべてOFFにして、指の制御だけ有効にしてください。


入力機器と指の紐づけ(キーバインド)

入力機器での指の曲げ伸ばしでは、設定ファイルによりキーやボタンと指の紐づけを調整することができます。
設定ファイルは以下の場所に配置されています。
※メモ帳などテキストエディタで開き編集してください。

SimpleMotionTracker_Data/StreamingAssets/finger_control_config.json (※このリンク先はGitHubです)

motionSpeed は 指の曲げ伸ばしスピードの設定です。

バインドには5つのグループがあります。

defaultPoses：標準ポーズ。何もボタンを押していない状態。
gripPoses：握りポーズ。何もボタンを押していない状態。
joypad：ジョイパッドのボタンバインド。
keyboard：キーボードのキーバインド。
mouse：マウスのボタンバインド。

キーバインド設定のパラメータは以下のようになっています。

①code は、キーまたはボタンに割り当てられた固有の名前です。 ジョイパッド、 キーボード、 マウス それぞれのボタンにコードが割り当てられています。
※コード名はリンク先で確認してください。

②rate は指の曲げ率です。0.0が開いている状態、1.0が握っている状態です。

③finger は指の指定です。R0～R4が右手親指～右手小指、L0～L4が左手親指～左手小指となります。

キーバインドでは1つのキー、ボタンで、複数の指を同時に制御することができます。
設定を変更、保存したあと、SMTで読み込むには Use Finger Control With Input Devises をOFF/ONしてください。

注意点
各グループの最後の行は、行末にカンマを付けてはいけません。設定ファイルが正常に読み込めなくなります。

ジョイパッドとキーバインドとデバイスの改造でデータグローブの様な使い方も可能になります。
